## 📝 UPDATE MAIN.dart FOR FIREBASE HOSTING

Your Flutter app needs to initialize Firebase with the config. Here's what's needed:

```dart
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart'; // Generated by FlutterFire CLI
import 'services/firebase_service.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  
  // Initialize Firebase with the config
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  
  // Initialize custom Firebase service
  await FirebaseService().initialize();
  
  runApp(const MyApp());
}
```

**The firebase_options.dart is auto-generated by Flutter, but you can also set it manually with your config.**

## 📧 EMAIL INTEGRATION: Firestore + Brevo

### How Email Works (Complete Flow)

#### User Sign-Up
```
1. User enters email in Flutter app
2. Flutter calls Firebase Auth: auth.createUserWithEmailAndPassword()
3. Firebase Auth creates user account
4. Flutter saves user doc to Firestore: users/{uid}
5. Backend (Cloud Function) listens to new users
6. Backend sends confirmation email via Brevo
7. User receives email with verification link
8. User clicks link → Firebase marks email verified
9. User logs in → Gets JWT token
10. User can now trade
```

#### Trade Alert (Example)
```
1. Autonomous trading AI executes trade
2. Backend saves trade to Firestore: users/{uid}/trades/
3. Firestore listener notifies Frontend (real-time)
4. Frontend shows notification immediately
5. Backend also sends email via Brevo
6. User gets: Instant app alert + Email notification
```

### Setup Firestore + Email (Backend)

The backend already has this configured! Here's how it works:

```python
# Backend: app/services/mail_delivery_service.py

@app.post("/api/auth/register")
async def register(email: str, password: str):
    # 1. Firebase Auth creates user
    user = await firebase_auth.create_user(email, password)
    
    # 2. Create user doc in Firestore
    await db.collection('users').document(user.uid).set({
        'email': email,
        'createdAt': firestore.SERVER_TIMESTAMP,
        'emailVerified': False,
    })
    
    # 3. Send confirmation email
    await mail_service.send_email(
        to_email=email,
        subject='Verify Your Forex Companion Account',
        html_body=EMAIL_TEMPLATE_HTML,
    )
    
    return {'status': 'verification_sent'}
```

## 🔐 SECURITY: Firestore Rules for Direct Access

Your Firestore must have security rules for direct Frontend access:

```firestore
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Users collection - Read/Write only own data
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
      
      // Trading history subcollection
      match /trades/{tradeId} {
        allow read, write: if request.auth.uid == userId;
      }
      
      // Settings subcollection
      match /settings/{settingId} {
        allow read, write: if request.auth.uid == userId;
      }
    }
    
    // Public data - Read only for authenticated users
    match /public/{document=**} {
      allow read: if request.auth != null;
      allow write: if false;
    }
    
    // Market data - Read only
    match /market_data/{document=**} {
      allow read: if request.auth != null;
      allow write: if false;
    }
  }
}
```

**These rules are already configured in Backend/firestore.rules**

## 📊 FIRESTORE DIRECT ACCESS (From Frontend)

With Firebase Hosting + Firestore, your Flutter app can:

### Read User Data (Real-Time)
```dart
final FirebaseService _firebase = FirebaseService();

// Get user data once
final userData = await _firebase.firestore
    .collection('users')
    .doc(userId)
    .get();

// Listen to changes (real-time)
_firebase.firestore
    .collection('users')
    .doc(userId)
    .snapshots()
    .listen((doc) {
      print('User data updated: ${doc.data()}');
    });
```

### Write Trade Data
```dart
// Save a new trade
await _firebase.firestore
    .collection('users')
    .doc(userId)
    .collection('trades')
    .add({
      'pair': 'EUR/USD',
      'amount': 1000,
      'price': 1.0850,
      'timestamp': Timestamp.now(),
    });
```

### Listen to Trades (Real-Time Alerts)
```dart
// Watch for new trades
_firebase.firestore
    .collection('users')
    .doc(userId)
    .collection('trades')
    .orderBy('timestamp', descending: true)
    .snapshots()
    .listen((snapshot) {
      for (var change in snapshot.docChanges) {
        if (change.type == DocumentChangeType.added) {
          // New trade! Show notification
          _showTradeAlert(change.doc.data());
        }
      }
    });
```

## 📧 EMAIL TRIGGERS (Backend Cloud Functions)

When data changes in Firestore, Backend can automatically send emails:

```python
# Backend: Would use Firebase Cloud Functions
# Trigger: On new trade in users/{uid}/trades

@functions.firestore_on_document_created('users/{uid}/trades/{tradeId}')
async def send_trade_email(event, context):
    trade_data = event.data.get()
    user_id = context.params['uid']
    
    # Get user email from Firestore
    user = await db.collection('users').doc(user_id).get()
    
    # Send email
    await mail_service.send_email(
        to_email=user['email'],
        subject=f"Trade Executed: {trade_data['pair']}",
        html_body=format_trade_email(trade_data),
    )
```
