# Security Onboarding Checklist

## Before Day 1

- [ ] Access to repository with least privilege
- [ ] Access to docs and runbooks
- [ ] Access to incident escalation contacts

## Week 1

- [ ] Install git hooks via `scripts/install-git-hooks.ps1` or `.sh`
- [ ] Run config validator locally
- [ ] Review `docs/SECURITY_CHECKLIST.md`
- [ ] Review `docs/INCIDENT_RESPONSE.md`

## Week 2

- [ ] Complete one secure code review with mentor
- [ ] Demonstrate incident reporting path
- [ ] Run weekly healthcheck script
